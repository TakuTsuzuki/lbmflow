## 1. 問題定義と座標規約（本書の標準形）

- 領域: 単位立方体 [0,1]³。x: 蓋駆動方向、y: スパン方向、z: 鉛直（底 z=0、蓋 z=1）。
- 境界条件: 全 6 面 no-slip 壁。上面 z=1 のみ接線速度 (U, 0, 0) で運動（スパン方向 y=0, 1 も**剛体静止壁**。スパン周期ではない）。
- Re = U·L/ν（L = キャビティ辺長）。
- 対称面: y = 0.5（定常解は Re ≲ 1900 でこの面に関して鏡映対称。§6.5 参照）。
- 中心線:
  - **鉛直中心線**: (x, y) = (0.5, 0.5)、z ∈ [0,1] 上の **u/U**（蓋平行成分）
  - **水平中心線**: (y, z) = (0.5, 0.5)、x ∈ [0,1] 上の **w/U**(鉛直成分)
- 表の抽出座標は 2D T7 と同じ **Ghia et al. (1982) の 17 点**（u 線は Ghia の
  y 座標、w 線は Ghia の x 座標と同一値）。

## 4. 滑らかさチェック（取込み時必須検査の実施記録）

### 4.1 方法

Ghia Re=400 v(0.9063)=−0.23827 型の誤植（1 点だけ隣接点と不連続）を検出する。
非一様座標なので**2 階分割差分**を用いる:

```
s_i  = (f_{i+1} − f_i) / (x_{i+1} − x_i)            （区間勾配）
D2_i = (s_i − s_{i−1}) / (x_{i+1} − x_{i−1})        （内点 i の 2 階分割差分）
```

D2 は「隣接 3 点を通る 2 次補間の 2 次係数」に等しい。1 点の転記誤り ε は
D2 の 3 連続点に **±交互の大スパイク**（大きさ ~ε/h²）を作るため、
D2 列の符号・大きさの推移が物理的に単調・滑らかであることを確認する。
疑わしい点には、片側 3 点（i−1, i+1, i+2 または i−2, i−1, i+1）による
2 次補間予測値 Q(x_i) と実値の残差を追加検査する（両側のうち小さい方の
|残差| が ~0.005·U を大きく超える点を疑義点とする）。

> 実施上の注意: 本チェックは本セッションの制約（コード実行禁止）により
> **手計算（分割差分の筆算）**で行い、主要な値は二重に検算した。codex は
> テストデータ凍結時に下記 D2 列をスクリプトで再計算し、本書の値と
> （丸め ±最終桁で）一致することを確認すること。

## 6. codex への推奨（テスト化指針）

### 6.1 テスト構成の推奨

- **T15.5a（凍結第一弾・Re=1000 のみ）**: 立方体 N³（全域計算。対称半領域の
  ショートカットは使わない — 対称性はテストの検査対象にする）、
  上面 MovingWall {[U,0,0]}、他 5 面 BounceBack、TRT Λ=3/16、U=0.1、
  Re=1000 → ν = U(N−2)/1000。
- 参照: §3.2 表 A（u/U, 17 点）・表 B（w/U, 17 点）、§3.3 A&K 極値。
- 比較は 2D T7 と同じ流儀: セル中心値から中心線へ（必要なら 2 点線形補間で
  参照座標へ）射影し、RMS を 17 点（両端含む）で取る。
  極値は格子値でなく局所放物線フィットで推定して比較。

### 6.2 格子解像度の目安と安定性制約

- **グリッド Re 制約（T10 の実測指針）**: Re/(N−2) = U/ν ≲ 15。
  Re=1000 では **N=64 は 16.1 でわずかに超過**（TRT 3/16 なら動く可能性は
  高いが余裕なし）。**推奨は N ≥ 72（Re/L=14.3）**、標準を N=80（12.8）に
  置くのが安全。N=96（10.6）は #[ignore] のフル検証用。
- 収束: 定常判定 ε=1e-8（Δ=500step）または上限 ~500k step。対流時間
  L/U = (N−2)/U ≈ 780 step（N=80, U=0.1）なので、2D T7 より収束は遅い
  （3D の渦構造の立ち上がりに ~50–100 L/U）。

### 6.3 推奨許容帯（初期帯。実測凍結の前段として）

2D Ghia の経験則（T7: RMS ≤ 0.02·U（Re100/400）/0.03·U（Re1000）、N=129、
実測 ~0.005·U）と §5 の 3D ソース間分散から:

| 格子 | u 線 RMS | w 線 RMS | 極値 |
|---|---|---|---|
| N=64–80（標準スイート） | ≤ 0.030·U | ≤ 0.035·U | \|u_min\|, \|w_min\| は A&K 比 ±6%、w_max ±8%、位置 ±0.03L |
| N=96–128（#[ignore]） | ≤ 0.020·U | ≤ 0.025·U | ±3%、位置 ±0.02L |

根拠の内訳（概算）: 参照そのものの不確かさ ~1e-5（無視可）、LBM 側の
格子バイアス（§5 の Zhang 系列: N≈64 相当で極値 −3〜−4%、= プロファイル
RMS 寄与 ~0.005–0.010·U）、参照点への補間誤差（蓋境界層 4 点で D2~150 →
h=1/78 で ~0.003·U）、O(Ma²) 誤差（U=0.1 で ~0.003·U）。合算 RMS 期待値は
0.008–0.015·U 程度で、上表は 2〜3 倍の余裕を持つ。2D の実績
（帯 0.02 に対し実測 0.005）と同じ余裕率である。
**w 線の帯を u 線より広めにしたのは**、w_min（−0.435）が u_min（−0.280）より
深く格子バイアスの絶対値が大きいこと、抽出点 x=0.9063 がほぼ極値上に
乗っていることによる。

### 6.4 追加で入れる価値のある assert（安価・高感度）

1. **対称性**: 定常解の対称面上スパン速度 max|v(x, y=mid, z)| ≤ 1e-3·U
   （物理的にゼロ。破れは未収束か対称性バグの検出器。N 偶数だと mid は
   セル間になるので隣接 2 面の平均で評価）。
2. **2D 混同検出**: 表 A と 2D Ghia Re=1000 の u 表の RMS 差は概算 ~0.1·U
   （最大点差 ~0.21、u_min: 3D −0.280 vs 2D −0.383）。「3D 解が誤って 2D 表に
   一致したら失敗」という逆向き assert（RMS vs 2D Ghia ≥ 0.05·U）を入れると、
   z 方向の実装が死んでいて実質 2D になる回帰を検出できる。
3. **収束傾向**: |u_min(N=96) − A&K| < |u_min(N=64) − A&K|（T8 と同じ流儀）。
4. **質量保存**: 全壁 BB 箱なので総質量ドリフト ≤ 1e-11 相対 /10⁴step（T6 同様）。

### 6.5 適用範囲の警告（物理）

- **定常性の上限**: 立方体（全壁）での定常–振動遷移は Re_cr ≈ 1914
  （Feldman & Gelfgat 2010; Liberzon et al. arXiv:1107.0449 の PIV 実験で追認、
  ω≈0.575 — 本文精読で確認。なお Kuhlmann & Albensoeder 2014 の線形安定性値
  ≈1919.5 は nekStab リポジトリ例題 README 経由の孫引きで、原本未確認）。
  **Re=1000 は定常域で安全**だが、2D の感覚で Re=2000 のケースを足すと
  臨界直上に乗るので不可。Re を増やすなら 1500 程度まで。
- **スパン周期境界との混同禁止**: スパン周期のキャビティは Re=1000 で既に
  Taylor–Görtler 型不安定域にあり、A&K の周期境界列（PICT に同梱）は
  2D 解でも「z 不変 3D 解」でもない**セル状 3D 定常解**。z 周期で走らせた
  D3Q19 が対称初期値から 2D 解に留まるか TGL セルに落ちるかは丸め摂動依存
  になり、テストとして不定。T15.5 は**全壁（剛体端壁）構成のみ**とする。
- **蓋角の正則化差**: 参照間でも蓋端特異点の扱いが異なる（A&K は特異性を
  解析的に処理、Ku/Zhang は蓋隣接ノードに u=0.3 のランプ）。LBMFlow の
  half-way BB 移動壁＋「速い壁が勝つ」コーナー規則はまた別の離散正則化で、
  その差は z ≥ 0.95 の 4 点に集中して現れる。キャリブレーション時は
  RMS を「全 17 点」と「蓋境界層 4 点を除く 13 点」の両方でログすること
  （仕様の帯は全 17 点に対して適用）。

### 6.6 データ埋め込み時の作法

- テストファイルには表 A/B と A&K 極値を定数として埋め込み、コメントで
  「Albensoeder & Kuhlmann, JCP 206 (2005) 536–558, Tables 5/6;
  transcribed via github.com/tum-pbs/PICT tests/validations.py @a95d7f9;
  smoothness-checked: docs/T15_5_CAVITY3D_REFERENCE.md §4」を明記する
  （T7 の Ghia 表と同じ運用。短い数表は事実データであり著作権上も問題ない）。
- 取込みスクリプトで §4 の D2 列を再計算し本書の値と照合（±最終桁）して
  から凍結すること（手計算の独立検証を兼ねる）。
