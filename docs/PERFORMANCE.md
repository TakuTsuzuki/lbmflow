# PERFORMANCE.md — 実測性能と精度/速度トレードオフガイド

測定環境: Apple Silicon 18 コア（P6+E12 相当）、macOS、rustc 1.93、
`--release`（thin LTO, codegen-units=1）。測定: `examples/bench_mlups.rs`。
MLUPS = 百万格子点更新/秒。2026-07-05 時点（偏差格納方式導入後）。

## 読み取れること（= 推奨設定）

1. **TRT は BGK と同速**（差は測定誤差内）。精度・安定性で優る TRT を常に
   デフォルトにしてよい。BGK は教育・比較用。
2. **f32 と f64 の速度差は ~5%**（現実装はメモリ帯域でなく演算で律速）。
   f32 の主な価値は**メモリ半減**（1024² で ~150MB→75MB、キャッシュ効率）と
   WASM/GUI 用途。
3. **偏差格納方式**（f − w を保持、2026-07-05 導入）により f32 の精度が劇的に改善:
   - 一様力の運動量成長誤差: 1.34e-3 → **2.8e-7**（~4800 倍改善）
   - TGV L2 誤差 (N=64): f32 7.1e-4 vs f64 7.0e-4（実質同等 = 検証グレード）
   → **通常の 2D シミュレーションは f32 で十分**。長時間積分・極小勾配・
   検証計算は f64。
4. シングルスレッド ~35 MLUPS → WASM GUI（256×128、60fps）で
   ~15-20 step/frame が現実的（docs/WASM_BRIDGE_DESIGN.md の想定通り）。

## 既知の最適化余地（Phase 7 バックログ）

- 現状は compute-bound: AoS cell-major + スカラーコード。SoA 化 + 明示 SIMD
  （or portable_simd）で f32 2-3 倍の余地。
- collide と stream の融合（1 パス化）でメモリトラフィック ~40% 削減余地。
- GPU（wgpu compute）で 10-100 倍級。3D 化とセットで検討。
