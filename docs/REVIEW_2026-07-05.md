## 2. 弱点・技術的負債（正直に）

1. **性能は「良い」が「最高」ではない**: compute-bound（f32 と f64 の差 5%）。
   AoS スカラーのまま。SoA + SIMD で 2-3 倍、GPU で 10-100 倍の余地。
2. **2D のみ**: 商用の本命用途（3D）が未着手。D3Q19 はコア設計の一般化が必要。
3. **混相流の制約**: 単成分のみ（二成分 MCMP 未実装 → RT 不安定性 T12 未達）。
   密度比 ~16（CS-EOS で 100+ に上げられる）。接触角の湿潤側レンジが狭い。
4. **開境界の品質**: ゼロ勾配 outflow は圧力反射あり（ratio 11）。Zou-He 出口の
   スタッガード層（減衰 4 セル）。characteristic BC で両方改善可能。
5. **円柱は staircase 近似**: Schäfer-Turek の帯が広い（±6%）。曲面境界
   （interpolated bounce-back）で ±1-2% に締められる。
6. **GUI の消しゴムはフロー再構築**（流れが止まる）。core に clear_solid があれば
   ライブ編集できる。単位変換（物理単位⇔格子単位）も未提供で、初学者には
   「ν=0.02 って何?」が残る。
7. **MCP は同期実行のみ**（長い run はブロック）。進捗通知・非同期ジョブ未対応。
8. **CI なし**（ローカルのみ）。リモートリポジトリが決まったら GitHub Actions を貼る。

## 3. 次期プラン（Phase 8-10）

### Phase 8: 物理の完成（混相流の約束を果たす）
- **8a. MCMP（二成分 Shan-Chen）**: 共有 solid/幾何で 2 つの分布関数を結合。
  検証: 相分離、液滴合体の質量保存、**RT 不安定性の線形成長率 ±20%（T12）**
- **8b. characteristic（対流）outlet**: T9 反射 ratio 11 → 3 以下へ。
  cylinder wake / チャネルで codex 検証
- **8c. 仮想壁密度**: 接触角 30-150° のフルレンジ制御（T11b 拡張）
- 検証は codex 発注（order #5）

### Phase 9: 性能リープ（トレードオフの「解決」を名乗る水準へ）
- **9a. SoA レイアウト + 明示 SIMD**（std::simd or wide）: f32 で 2-3 倍
- **9b. 融合 collide-stream**（メモリトラフィック 40% 減）
- **9c.（評価）wgpu compute バックエンド**: 同一 Simulation API の後ろに
  GPU 実装。2D で 10-50 倍、3D の前提技術
- ベンチ表を PERFORMANCE.md で更新、GUI の解像度上限を引き上げ

### Phase 10: 3D（D3Q19）
- lattice を定数ジェネリクスで抽象化（D2Q9 / D3Q19）
- 検証: 3D TGV、3D Poiseuille、球の抗力
- GUI: 断面表示 + 等値面（or 3D は CLI/エージェント専用と割り切る判断も可）

### 随時（小粒・高価値）
- core: clear_solid（GUI ライブ編集）、単位変換ヘルパ（UnitConverter）
- GUI: 単位表示モード、シナリオ書き出しボタン（GUI→Agent モード連携）
- CLI: VTK 出力、`lbm gallery`（全プリセット一括実行 → HTML レポート）
- MCP: 非同期 run + 進捗通知
- リモートリポジトリ + CI（GitHub Actions: cargo test + wasm + web build）

## 4. 実行順の判断
